What is the Zero Trust Migration Journey Add-in?

The Zero Trust Migration Journey Add-in is a Windows desktop dashboard that helps IT teams visualize and manage their ConfigMgr to Microsoft Intune migration. It connects to your Microsoft Graph and ConfigMgr environment to provide real-time insights on device enrollment progress and health.

Key Features

  • Real-Time Data: Connects to Microsoft Graph and ConfigMgr Admin Service to pull actual device data
  • Enrollment Tracking: Visualize enrollment progress with charts, velocity metrics, and trend analysis
  • Device Readiness Analysis: 4-tier health scoring (Excellent/Good/Fair/Poor) with predicted success rates
  • Smart Enrollment Management: AI-powered device categorization and enrollment recommendations
  • Enrollment Momentum: Track velocity, set targets, and project migration timelines
  • Device Identity Analysis: Understand Hybrid Entra, Entra-joined, and AD-only device states
  • Autonomous Enrollment Agent: Optional AI agent for automated enrollment with configurable risk tolerance
  • Automatic Updates: Self-updating application ensures you always have the latest features
  • Privacy-Safe Telemetry: Anonymous usage analytics help improve the tool

Current Focus: Migration Dashboard

This release provides comprehensive migration tracking across multiple dashboard tabs:

  • Overview Tab: Executive summary with device counts, enrollment status, device identity breakdown, and compliance scorecard
  • Enrollment Tab: Smart Enrollment Management with 4-tier device readiness, enrollment momentum tracking, and optional AI agent
  • Workloads Tab: Track workload migration from ConfigMgr to Intune (hidden by default)
  • AI Actions Tab: View AI-generated recommendations and blockers (hidden by default)

üöÄ Automatic Updates

How It Works

The Zero Trust Migration Journey Add-in updates itself automatically - no manual downloads or IT deployment needed:

Stage What Happens Duration
1. Check for Updates Every launch, checks GitHub for newer versions 2-4 seconds
2. Download Delta Only downloads changed files (80-90% bandwidth savings) 20-30 seconds
3. UAC Prompt Windows requests admin elevation to update files Click "Yes"
4. Apply Update Files replaced automatically in background 5-10 seconds
5. Restart App relaunches with new version Immediate

Common Questions

Why do I see a UAC prompt?

The app is installed in Program Files, which requires administrator privileges to modify. This is standard Windows security - the same prompt you see when updating Chrome, VS Code, or other apps.

What if I click "No" on the UAC prompt?

The update won't install and you'll continue using the current version. The update will be offered again next time you launch the app.

What if I'm offline?

The update check will timeout after 10 seconds and the app will continue loading with your current version. No errors or delays.

Can I disable updates?

Updates are always enabled to ensure you have the latest security fixes and features. The process is fast (30-45 seconds) and only happens when launching.

‚òÅÔ∏è Cloud Readiness Signals

Cloud Readiness Signals help you assess your organization's readiness for various cloud migration scenarios. Each signal evaluates specific requirements and identifies blockers that may prevent successful migration.

Data Sources

Cloud Readiness assessments combine data from multiple sources:

  • ConfigMgr (Admin Service/WMI): Device inventory, TPM status, OS details, hardware specifications
  • Microsoft Graph API: Intune enrollment status, Autopilot registration, Entra ID join status, compliance data
üí° Important: The "Total Devices" count for most signals comes from ConfigMgr, as these are the devices you're planning to migrate. Intune/Graph data is used to cross-reference enrollment and registration status.

üöÄ Autopilot Registration

What it measures: ConfigMgr devices that are registered to Windows Autopilot for Zero-Touch provisioning.

Metric Description Data Source
Total Devices Windows 10/11 devices in ConfigMgr ConfigMgr Admin Service
Ready Devices Devices registered to Autopilot AND meeting OS requirements Graph API (WindowsAutopilotDeviceIdentities)

Blockers Checked:

  • Not Registered to Autopilot (Medium): Devices not yet in Autopilot. Register via hardware hash upload or OEM.
  • Unsupported OS Version (High): Devices running Windows 10 older than version 1809 (build 17763).
‚ö†Ô∏è Note: TPM 2.0 is NOT required for Autopilot registration. TPM 2.0 is required for Windows 11, BitLocker, and Windows Hello for Business, but devices can register with Autopilot without TPM 2.0.

ü™ü Windows 11 Readiness

What it measures: Windows 10 devices that meet the hardware requirements for Windows 11 upgrade.

Metric Description Data Source
Total Devices Windows 10 devices in ConfigMgr (excludes devices already on Windows 11) ConfigMgr Admin Service
Ready Devices Windows 10 devices meeting all Windows 11 hardware requirements ConfigMgr Hardware Inventory

Blockers Checked:

  • Missing TPM 2.0 (Critical): TPM 2.0 is required for Windows 11. Enable in BIOS or upgrade hardware.
  • UEFI/Secure Boot (High): Windows 11 requires UEFI firmware with Secure Boot enabled.
üí° Tip: If you see 0% readiness but have devices, check if TPM hardware inventory is enabled in ConfigMgr. Devices without TPM data are counted as "not ready."

‚òÅÔ∏è Cloud-Native Readiness

What it measures: Progress toward full cloud-native management for devices currently managed by ConfigMgr. The goal is to move all workloads to Microsoft Intune so the ConfigMgr agent can be safely removed.

üìä Assessment Scope: This signal assesses only devices with a ConfigMgr record (your migration targets). "Born-in-cloud" devices (Entra + Intune with no ConfigMgr history) are already cloud-native and are excluded from this assessment.

A ConfigMgr device is "Cloud-Native Ready" when:

  • ‚úÖ Co-Managed with ALL workloads moved to Microsoft Intune

This means all 8 co-management workload sliders are set to Intune, allowing the ConfigMgr client to be safely removed.

Workload Device Drill-Down

Click the "X devices" link next to the "Co-Managed with Workloads on ConfigMgr" blocker to see a detailed view of which workloads are on ConfigMgr vs. Intune for each device. The dialog shows:

  • ‚úÖ Green checkmarks - Workload managed by Intune (good)
  • ‚öôÔ∏è Orange gears - Workload still on ConfigMgr (needs migration)

Use this to identify which workload sliders need to be moved in the ConfigMgr console.

Co-Management Workload Authority

For co-managed devices, the dashboard queries the Graph API to determine which workloads are managed by ConfigMgr vs. Intune. A device with ALL 8 workloads on Intune is considered "cloud-native ready" because the ConfigMgr agent can be safely removed.

Data Sources

üîß Troubleshooting

Common Issues

Issue: "Can't connect to Microsoft Graph"

Possible Causes:

  • Insufficient Azure AD permissions
  • Network firewall blocking graph.microsoft.com
  • Expired authentication token

Solutions:

  1. Verify you have Intune Administrator or higher role
  2. Check firewall allows HTTPS to *.microsoft.com
  3. Click "Connect to Microsoft Graph" to re-authenticate

Issue: "Need admin approval" or "Authorization_RequestDenied"

Cause: The app uses Microsoft Graph permissions that require admin consent before any user in your tenant can use them. This is a one-time setup step.

Solutions:

  1. Option 1 - Sign in as Global Admin:
    • Click "Have an admin account? Sign in with that account"
    • Sign in with a Global Administrator account
    • Grant consent when prompted ("Consent on behalf of your organization")
    • After consent is granted, you can use your regular Intune admin account
  2. Option 2 - Pre-approve in Azure Portal:
    • Go to Azure Portal ‚Üí Enterprise Applications
    • Search for "Microsoft Graph Command Line Tools"
    • Click Permissions ‚Üí Grant admin consent for [Your Tenant]
    • All users with Intune roles can now use the app
üí° Note: Admin consent only needs to be granted once per tenant. After consent, all users with appropriate Intune roles (Intune Administrator, Global Administrator, etc.) can sign in without seeing this error.

Issue: App won't launch

Solutions:

  1. Check Windows Event Viewer for .NET errors
  2. Ensure .NET 8 runtime isn't blocked by Group Policy
  3. Try running as administrator
  4. Check logs in %LOCALAPPDATA%\ZeroTrustMigrationAddin\Logs\

Issue: Update fails with "Access Denied"

Solutions:

  1. Make sure you click "Yes" on the UAC prompt
  2. Check you have write permissions to the install folder
  3. Close any antivirus that might be blocking file updates

Issue: Data looks incorrect

Solutions:

  1. Click the refresh button to reload data
  2. Verify device counts match Intune portal
  3. Check if you're in Mock Data mode (status bar shows data source)

‚ùì Frequently Asked Questions

Q: Is this tool officially from Microsoft?

A: No, this is a community tool built to help IT teams manage their ConfigMgr to Intune migrations. It uses official Microsoft Graph APIs to pull data.

Q: What data does telemetry collect?

A: We collect anonymous usage analytics (which buttons you click, errors that occur) to improve the tool. We DO NOT collect usernames, device names, tenant IDs, emails, IPs, or any identifiable information. All PII is automatically sanitized before sending to Azure.

Q: Can I use this in production?

A: Yes! The tool reads data only by default - it doesn't modify your Intune or ConfigMgr environment. The Autonomous Enrollment Agent can make changes, but only when explicitly enabled.

Q: How often does data refresh?

A: Data refreshes when you click the refresh button or switch between data sources. The Microsoft Graph data is real-time.

Q: What's the difference between health score tiers?

A: Excellent (‚â•85) has 98% enrollment success rate, Good (60-84) has 85%, Fair (40-59) has 60%, and Poor (<40) has only 30%. Focus on Excellent/Good devices first for quick wins.

Q: Why are some devices showing as "Blocked"?

A: Devices in AD-only or Workgroup state cannot be enrolled in Intune. They need Hybrid Azure AD Join or Azure AD Join first. The Device Identity section shows which devices are blocked.

Q: Is the Autonomous Enrollment Agent safe to use?

A: The Autonomous Enrollment Agent is not yet functional - it's under development. The UI is visible for preview purposes, but no automated enrollments are being performed. When completed, it will use a ReAct architecture with configurable autonomy levels.

Q: Why do I need admin permissions for updates?

A: The app is installed in Program Files, which is protected by Windows. Admin elevation is required to modify files in that location - this is standard security practice.

Q: What happens to ConfigMgr integration?

A: ConfigMgr integration is automatic when the ConfigMgr console is installed. The tool uses the Admin Service API to pull device health data. If ConfigMgr isn't available, Graph-only mode works fine.

Q: How do I export device lists?

A: The Smart Enrollment Management section has an "Export Device List" button that exports categorized devices to CSV for remediation planning.

Q: Where are my Graph tokens stored?

A: Authentication tokens are managed by MSAL (Microsoft Authentication Library) and stored securely in Windows Credential Manager. They're never written to log files.

Q: Why do I see "Need admin approval" when signing in?

A: This app uses Microsoft Graph API permissions that require admin consent before any user can authenticate. A Global Administrator needs to grant consent once for your entire tenant. Either sign in with a Global Admin account and approve, or ask your Azure AD admin to pre-approve the "Microsoft Graph Command Line Tools" app in Azure Portal ‚Üí Enterprise Applications ‚Üí Permissions. Once consented, all users with Intune roles can sign in normally.

Q: Can I deploy this via Intune/SCCM?

A: Yes, but it's not required. The tool updates itself automatically from GitHub. You can deploy the initial install via Intune/SCCM, and users will receive automatic updates going forward.

Q: How do I show hidden tabs like Workloads or AI Actions?

A: Use the command-line argument: ZeroTrustMigrationAddin.exe /ShowOnlyTabs:Overview,Enrollment,Workloads,AI
